section ‚ÄπSumming lists in a monoid‚Ä∫

theory Monoid_ZF_1 imports Monoid_ZF

begin


text‚ÄπThis theory consider properties of sums of monoid elements, similar
  to the ones formalized in the ‚ÄπSemigroup_ZF‚Ä∫ theory for sums of semigroup elements. The main
  difference is that since each monoid has a neutral element it makes sense to define a sum
  of an empty list of monoid elements. In multiplicative notation the properties considered here
  can be applied to natural powers of elements ($x^n, n\in \mathbb{N}$) in group or ring theory or,
  when written additively, to natural multiplicities $n\cdot x, n\in \mathbb{N}$). ‚Ä∫

subsection‚ÄπNotation and basic properties of sums of lists of monoid elements‚Ä∫

text‚ÄπIn this section we setup a contex (locale) with notation for sums of lists of
  monoid elements and prove basic properties of those sums in terms of that notation. ‚Ä∫

text‚ÄπThe locale (context) ‚Äπmonoid1‚Ä∫ extends the locale ‚Äπmonoid1‚Ä∫, adding the notation for the 
  neutral element as $0$ and the sum of a list of monoid elements.
  It also defines a notation for natural multiple of an element of a monoid,
  i.e. $n\cdot x = x\oplus x\oplus ... \oplus x$ (n times). ‚Ä∫

locale monoid1 = monoid0 +
  fixes mzero ("ùü¨")
  defines mzero_def [simp]: "ùü¨ ‚â° TheNeutralElement(G,f)"

  fixes listsum ("‚àë _" 70)
  defines listsum_def [simp]: "‚àës ‚â° Fold(f,ùü¨,s)"

  fixes nat_mult (infix "‚ãÖ" 72)
  defines nat_mult_def [simp]: "n‚ãÖx ‚â° ‚àë{‚ü®k,x‚ü©. k‚ààn}"

text‚ÄπLet's recall that the neutral element of the monoid is an element of the monoid (carrier) $G$
  and the monoid operation ($f$ in our notation) is a function that maps $G\times G$
  to $G$.‚Ä∫

lemma (in monoid1) zero_monoid_oper: shows "ùü¨‚ààG" and "f:G√óG ‚Üí G"
  using monoidAssum unit_is_neutral unfolding IsAmonoid_def IsAssociative_def 
  by simp_all

text‚ÄπThe sum of a list of monoid elements is a monoid element.‚Ä∫

lemma (in monoid1) sum_in_mono: assumes "n‚àànat" "‚àÄk‚ààn. q(k)‚ààG"
  shows "(‚àë{‚ü®k,q(k)‚ü©. k‚ààn}) ‚àà G"
proof -
  let ?a = "{‚ü®k,q(k)‚ü©. k‚ààn}"
  from assms have "n ‚àà nat" "f:G√óG ‚Üí G" "?a:n ‚Üí G" "ùü¨‚ààG" "G‚â†0"
    using zero_monoid_oper ZF_fun_from_total by auto
  then show ?thesis using fold_props by simp
qed

text‚ÄπThe reason we start from $0$ in the definition of the summation sign in the ‚Äπmonoid1‚Ä∫ locale
  is that we want to be able to sum the empty list. Such sum of the empty list is $0$. ‚Ä∫

lemma (in monoid1) sum_empty: assumes "s:0‚ÜíG" shows "(‚àës) = ùü¨"
  using assms zero_monoid_oper fold_empty group0_1_L3A by simp

text‚ÄπFor nonempty lists our $\Sigma$ is the same as ‚ÄπFold1‚Ä∫. ‚Ä∫

lemma (in monoid1) sum_nonempty:  assumes "n ‚àà nat" "s:succ(n)‚ÜíG"
  shows 
    "(‚àës) = Fold(f,s`(0),Tail(s))"
    "(‚àës) = Fold1(f,s)"
proof -
  from assms have "s`(0) ‚àà G" using empty_in_every_succ apply_funtype 
    by simp
  with assms have "(‚àës) = Fold(f,ùü¨‚äïs`(0),Tail(s))"
    using zero_monoid_oper fold_detach_first by simp
  with ‚Äπs`(0) ‚àà G‚Ä∫ show "(‚àës) = Fold(f,s`(0),Tail(s))"
    using unit_is_neutral by simp
  then show "(‚àës) = Fold1(f,s)" unfolding Fold1_def by simp
qed

text‚ÄπWe can pull the first component of a sum of a nonempty list of monoid elements
  before the summation sign. ‚Ä∫  

lemma (in monoid1) seq_sum_pull_first0: assumes "n ‚àà nat" "s:succ(n)‚ÜíG"
  shows "(‚àës) = s`(0) ‚äï (‚àëTail(s))"
proof -
  from assms have "s`(0) ‚àà G" using empty_in_every_succ apply_funtype 
    by simp
  { assume "n=0"
    with assms have "Tail(s):0‚ÜíG" using tail_props(1) by blast
    with assms ‚Äπs`(0) ‚àà G‚Ä∫ have "(‚àës) = s`(0) ‚äï (‚àëTail(s))"
      using sum_nonempty(1) sum_empty zero_monoid_oper(2) group0_1_L3A 
        fold_empty unit_is_neutral by simp
  }
  moreover
  { assume "n‚â†0"
    with assms(1) obtain m where "m‚àànat" and "n = succ(m)"
      using Nat_ZF_1_L3 by blast
    with assms have "Tail(s):succ(m)‚ÜíG" using tail_props(1) 
      by simp
    let ?a = "{‚ü®0,s`(0)‚ü©}"
    from ‚Äπs`(0) ‚àà G‚Ä∫ have "0‚àànat" "?a:succ(0)‚ÜíG"
      using pair_func_singleton succ_explained by simp_all
    with ‚Äπm‚àànat‚Ä∫ ‚ÄπTail(s):succ(m)‚ÜíG‚Ä∫ 
    have "f`‚ü®Fold1(f,?a),Fold1(f,Tail(s))‚ü© = Fold1(f,Concat(?a,Tail(s)))"
      using semigr0_valid_in_monoid0 semigr0.prod_conc_distr
      by blast
    with assms ‚Äπ?a:succ(0)‚ÜíG‚Ä∫ ‚Äπm‚àànat‚Ä∫ ‚ÄπTail(s):succ(m)‚ÜíG‚Ä∫ 
    have "(‚àës) = s`(0) ‚äï (‚àëTail(s))" 
      using semigr0_valid_in_monoid0 semigr0.prod_of_1elem pair_val 
        sum_nonempty(2) first_concat_tail by simp
  }
  ultimately show ?thesis by auto
qed

text‚ÄπThe first assertion of the next theorem is similar in content to ‚Äπseq_sum_pull_first0‚Ä∫ 
  formulated in terms of the expression defining the list of monoid elements. The second
  one shows the dual statement: the last element of a sequence can be pulled out of the
  sequence and put after the summation sign. So, we are showing here that 
  $\sum_{k=0}^{n} q_k = q_0 \oplus \sum_{k=0}^{n-1} q_{k+1} = (\sum_{k=0}^{n-1} q_k) \oplus q_n. $ ‚Ä∫

theorem (in monoid1) seq_sum_pull_one_elem: 
  assumes "n ‚àà nat" "‚àÄk‚ààn #+ 1. q(k) ‚àà G"
  shows 
    "(‚àë{‚ü®k,q(k)‚ü©. k‚ààn #+ 1}) = q(0) ‚äï (‚àë{‚ü®k,q(k #+ 1)‚ü©. k‚ààn})"
    "(‚àë{‚ü®k,q(k)‚ü©. k‚ààn #+ 1}) = (‚àë{‚ü®k,q(k)‚ü©. k‚ààn}) ‚äï q(n)"
proof -
  let ?s = "{‚ü®k,q(k)‚ü©. k‚ààn #+ 1}"
  from assms(1) have "0 ‚àà n #+ 1" using empty_in_every_succ succ_add_one(1)
    by simp
  then have "?s`(0) = q(0)" by (rule ZF_fun_from_tot_val1)
  from assms(2) have "?s: n #+ 1 ‚Üí G" by (rule ZF_fun_from_total)
  with assms(1) ‚Äπ?s`(0) = q(0)‚Ä∫ have "(‚àë?s) = q(0) ‚äï (‚àëTail(?s))"
    using seq_sum_pull_first0 by simp
  moreover from assms have "Tail(?s) = {‚ü®k,q(k #+ 1)‚ü©. k ‚àà n}"
    using tail_formula by simp
  ultimately show "(‚àë{‚ü®k,q(k)‚ü©. k‚ààn #+ 1}) =  q(0) ‚äï (‚àë{‚ü®k,q(k #+ 1)‚ü©. k‚ààn})" 
    by simp
  from assms show "(‚àë{‚ü®k,q(k)‚ü©. k‚ààn #+ 1}) =  (‚àë{‚ü®k,q(k)‚ü©. k‚ààn}) ‚äï q(n)"
    using zero_monoid_oper fold_detach_last by simp
qed

text‚ÄπThe sum of a singleton list is its only element,‚Ä∫

lemma (in monoid1) seq_sum_singleton: assumes "q(0) ‚àà G"
  shows "(‚àë{‚ü®k,q(k)‚ü©. k‚àà1}) = q(0)"
proof -
  from assms have "0‚àànat" and "‚àÄk‚àà0 #+ 1. q(k) ‚àà G" by simp_all
  then have 
    "(‚àë{‚ü®k,q(k)‚ü©. k‚àà0 #+ 1}) = q(0) ‚äï (‚àë{‚ü®k,q(k #+ 1)‚ü©. k‚àà0})"
    by (rule seq_sum_pull_one_elem)
  with assms show ?thesis using sum_empty unit_is_neutral by simp
qed

text‚ÄπIf the monoid operation is commutative, then the sum of a nonempty sequence
  added to another sum of a nonempty sequence of the same length is equal 
  to the sum of pointwise sums of the sequence elements. 
  This is the same as the theorem ‚Äπprod_comm_distrib‚Ä∫ from the 
  ‚ÄπSemigroup_ZF‚Ä∫ theory, just written in the notation used in the ‚Äπmonoid1‚Ä∫ locale.‚Ä∫

lemma (in monoid1) sum_comm_distrib0:
  assumes  "f {is commutative on} G" "n‚àànat" and
  "a : n #+ 1 ‚Üí G"  "b : n #+ 1 ‚Üí G"  "c : n #+ 1 ‚Üí G" and
  "‚àÄj‚ààn #+ 1. c`(j) = a`(j) ‚äï b`(j)"
  shows "(‚àë c) = (‚àë a) ‚äï (‚àë b)"
  using assms succ_add_one(1) sum_nonempty 
    semigr0_valid_in_monoid0 semigr0.prod_comm_distrib by simp

text‚ÄπAnother version of ‚Äπsum_comm_distrib0‚Ä∫ written in terms of the expressions
  defining the sequences, shows that for commutative monoids we have 
  $\sum_{k=0}^{n-1}q(k) \oplus p(k) = (\sum_{k=0}^{n-1} p(k))\oplus (\sum_{k=0}^{n-1} q(k))$. ‚Ä∫

theorem (in monoid1) sum_comm_distrib: 
  assumes  "f {is commutative on} G" "n‚àànat" and
  "‚àÄk‚ààn. p(k) ‚àà G" "‚àÄk‚ààn. q(k) ‚àà G"
  shows 
    "(‚àë{‚ü®k,p(k)‚äïq(k)‚ü©. k‚ààn}) = (‚àë{‚ü®k,p(k)‚ü©. k‚ààn}) ‚äï (‚àë{‚ü®k,q(k)‚ü©. k‚ààn})" 
proof -
  let ?a = "{‚ü®k,p(k)‚ü©. k‚ààn}"
  let ?b = "{‚ü®k,q(k)‚ü©. k‚ààn}"
  let ?c = "{‚ü®k,p(k)‚äïq(k)‚ü©. k‚ààn}"
  { assume "n=0"
    then have "(‚àë?c) = (‚àë?a) ‚äï (‚àë?b)"
      using sum_empty unit_is_neutral by simp
  }
  moreover
  { assume "n‚â†0"
    with assms(2) obtain m where "m‚àànat" and  "n = m #+ 1"
      using nat_not0_succ by blast
    from assms(3,4) have "?a:n‚ÜíG" "?b:n‚ÜíG" "?c:n‚ÜíG" 
      using group0_1_L1 ZF_fun_from_total by simp_all
    with assms(1) ‚Äπm‚àànat‚Ä∫ ‚Äπn = m #+ 1‚Ä∫ have 
      "f {is commutative on} G" "m‚àànat" and
      "?a:m #+ 1‚ÜíG" "?b:m #+ 1‚ÜíG" "?c:m #+ 1‚ÜíG"
      by simp_all
    moreover have "‚àÄk‚ààm #+ 1. ?c`(k) = ?a`(k) ‚äï ?b`(k)"
    proof -
      { fix k assume "k ‚àà m #+ 1"
        with ‚Äπn = m #+ 1‚Ä∫ have "k‚ààn" by simp
        then have "?c`(k) = ?a`(k) ‚äï ?b`(k)"
          using ZF_fun_from_tot_val1 by simp_all
      } thus ?thesis by simp
    qed
    ultimately have "(‚àë?c) = (‚àë?a) ‚äï (‚àë?b)"
      using sum_comm_distrib0 by simp
  }
  ultimately show ?thesis by blast
qed

subsection‚ÄπMultiplying monoid elements by natural numbers‚Ä∫

text‚ÄπA special case of summing (or, using more notation-neutral term ‚Äπfolding‚Ä∫)
  a list of monoid elements is taking a natural multiple of a single element. 
  This can be applied to various monoids embedded in other algebraic structures.
  For example a ring is a monoid with addition as the operation, so the notion of 
  natural multiple directly transfers there. Another monoid in a ring is formed by its 
  multiplication operation. In that case the natural multiple maps into natural powers of a ring
  element. ‚Ä∫

text‚ÄπThe zero's multiple of a monoid element is its neutral element.‚Ä∫

lemma (in monoid1) nat_mult_zero: shows "0‚ãÖx = ùü¨" using sum_empty by simp

text‚ÄπAny multiple of a monoid element is a monoid element.‚Ä∫

lemma (in monoid1) nat_mult_type: assumes "n‚àànat" "x‚ààG"
  shows "n‚ãÖx ‚àà G" using assms sum_in_mono by simp

text‚ÄπTaking one more multiple of $x$ adds $x$. ‚Ä∫

lemma (in monoid1) nat_mult_add_one: assumes "n‚àànat" "x‚ààG" 
  shows "(n #+ 1)‚ãÖx = n‚ãÖx ‚äï x" and "(n #+ 1)‚ãÖx = x ‚äï n‚ãÖx"
proof -
  from assms(2) have I: "‚àÄk‚ààn #+ 1. x ‚àà G" by simp
  with assms(1) have "(‚àë{‚ü®k,x‚ü©. k ‚àà n #+ 1}) = x ‚äï (‚àë{‚ü®k,x‚ü©. k‚ààn})"
    by (rule seq_sum_pull_one_elem)
  thus "(n #+ 1)‚ãÖx = x ‚äï n‚ãÖx" by simp
  from assms(1) I have "(‚àë{‚ü®k,x‚ü©. k‚ààn #+ 1}) =  (‚àë{‚ü®k,x‚ü©. k‚ààn}) ‚äï x"
    by (rule seq_sum_pull_one_elem)
  thus "(n #+ 1)‚ãÖx = n‚ãÖx ‚äï x" by simp
qed

text‚ÄπOne element of a monoid is that element.‚Ä∫

lemma (in monoid1) nat_mult_one: assumes "x‚ààG" shows "1‚ãÖx = x"
proof -
  from assms have "(0 #+ 1)‚ãÖx = 0‚ãÖx ‚äï x" using nat_mult_add_one(1) by blast
  with assms show ?thesis using nat_mult_zero unit_is_neutral by simp
qed

text‚ÄπMultiplication of $x$ by a natural number induces a homomorphism between natural numbers 
  with addition and and the natural multiples of $x$. ‚Ä∫

lemma (in monoid1) nat_mult_add: assumes "n‚àànat" "m‚àànat" "x‚ààG"
  shows "(n #+ m)‚ãÖx = n‚ãÖx ‚äï m‚ãÖx"
proof -
  from assms have "m‚àànat" and "(n #+ 0)‚ãÖx = n‚ãÖx ‚äï 0‚ãÖx" 
    using nat_mult_type unit_is_neutral nat_mult_zero by simp_all
  moreover from assms(1,3) have
    "‚àÄk‚àànat. (n #+ k)‚ãÖx = n‚ãÖx ‚äï k‚ãÖx ‚ü∂ (n #+ (k #+ 1))‚ãÖx = n‚ãÖx ‚äï (k #+ 1)‚ãÖx"
     using nat_mult_type nat_mult_add_one(1) sum_associative by simp
  ultimately show ?thesis by (rule ind_on_nat1)
qed
